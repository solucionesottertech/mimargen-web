# SEO Foundation Specification

## Purpose

Defines the SEO and accessibility baseline: semantic HTML structure, meta/OG/Twitter tags, JSON-LD structured data, sitemap, robots.txt, and accessible markup. Targets Lighthouse SEO â‰Ą 95.

## Requirements

| ID | Requirement | Priority |
|----|-------------|----------|
| SE-01 | Every page MUST have `<title>`, `<meta name="description">`, and `<link rel="canonical">` | Must |

| SE-03 | JSON-LD structured data MUST include `SoftwareApplication` for MiMargen and `FAQPage` for the FAQ section | Must |

### Requirement: Essential Meta Tags (SE-01)

Every page MUST include a unique `<title>` (â‰¤ 60 chars), a `<meta name="description">` (â‰¤ 160 chars), and a `<link rel="canonical">` pointing to the definitive URL. These MUST be injected via Astro's frontmatter or the `<Head>` component.

#### Scenario: Meta tags render in page head

- GIVEN the landing page loads
- WHEN inspecting `<head>` in the rendered HTML
- THEN `<title>` contains "MiMargen | Calcula el costo real de tus recetas"
- AND `<meta name="description">` is 120-160 characters of keyword-rich copy
- AND `<link rel="canonical" href="https://mimargen.com/">` is present

#### Scenario: Meta tags are unique per page

- GIVEN future pages are added (e.g., blog, pricing)
- WHEN rendering any page
- THEN its `<title>` and `<meta name="description">` are distinct from other pages
- AND its canonical URL points to its own path

### Requirement: Social Media Tags (SE-02)

Every page MUST include Open Graph (`og:title`, `og:description`, `og:image`, `og:url`, `og:type`) and Twitter Card (`twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`) tags. Default image must be a 1200Ă—630px PNG.

#### Scenario: OG tags render for social sharing

- GIVEN the landing page HTML
- WHEN scraped by Facebook/Twitter crawlers
- THEN `og:title` matches the page `<title>`
- AND `og:image` points to a valid 1200Ă—630px image URL
- AND `og:type` is `"website"`
- AND `twitter:card` is `"summary_large_image"`

#### Scenario: Fallback OG image exists

- GIVEN no section-specific OG image is set
- WHEN the page renders
- THEN `og:image` falls back to `https://mimargen.com/og-default.png`

### Requirement: JSON-LD Structured Data (SE-03)

Two JSON-LD blocks MUST be injected: `SoftwareApplication` on every page (describing MiMargen: name, description, applicationCategory, operatingSystem, offers) and `FAQPage` only on the FAQ section page.

#### Scenario: SoftwareApplication schema validates

- GIVEN the rendered HTML
- WHEN pasted into Google Rich Results Test
- THEN the `SoftwareApplication` schema is detected and passes validation
- AND includes `name`, `description`, `applicationCategory: "BusinessApplication"`, `operatingSystem: "Web"`

#### Scenario: FAQPage schema includes all questions

- GIVEN the FAQ section has 6 questions with answers
- WHEN the page renders
- THEN the `FAQPage` JSON-LD block contains a `mainEntity` array with 6 `Question` items
- AND each `Question` has an `acceptedAnswer.text`

### Requirement: Sitemap and Robots (SE-04)

`sitemap.xml` MUST be auto-generated by `@astrojs/sitemap` at build time, listing all pages with `<lastmod>` and `<priority>`. `robots.txt` MUST allow all crawlers and reference the sitemap URL.

#### Scenario: Sitemap is generated at build

- GIVEN `@astrojs/sitemap` is configured in `astro.config.mjs`
- WHEN `pnpm build` runs
- THEN `dist/sitemap-0.xml` (or `dist/sitemap.xml`) exists
- AND contains `<url>` entries for all site pages with valid `<loc>`

#### Scenario: robots.txt allows crawling

- GIVEN a static `robots.txt` in the `public/` directory or generated at build
- WHEN accessing `https://mimargen.com/robots.txt`
- THEN the response is:
  ```
  User-agent: *
  Allow: /
  Sitemap: https://mimargen.com/sitemap.xml
  ```

### Requirement: Image Alt Text and ARIA Labels (SE-05)

All `<img>` tags MUST have non-empty `alt` attributes describing the image content in Chilean Spanish. Icon-only buttons and links MUST have `aria-label`. Decorative images use `alt=""`.

#### Scenario: Every image has descriptive alt text

- GIVEN all rendered images on the landing page
- WHEN auditing `<img>` tags
- THEN no `<img>` has a missing `alt` attribute
- AND content images have descriptive alt text (e.g., "Calculadora de costos de recetas MiMargen")
- AND decorative icons use `alt=""`

#### Scenario: Icon-only CTAs have aria-label

- GIVEN a social media icon link in the Footer (no visible text)
- WHEN inspecting the element
- THEN it has `aria-label="SĂ­guenos en Instagram"` or equivalent

### Requirement: Heading Hierarchy Verification (SE-06)

The page heading structure MUST be programmatically verifiable: exactly one `<h1>`, no more than 20 `<h2>` elements, all `<h3>` preceded by an `<h2>`, and no skipped levels.

#### Scenario: Heading hierarchy passes automated audit

- GIVEN the rendered landing page HTML
- WHEN running an accessibility audit (axe-core or Lighthouse)
- THEN the "Heading levels should only increase by one" rule passes
- AND no "Page must contain a level-one heading" violations exist
